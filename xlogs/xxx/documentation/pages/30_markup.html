The items markup uses javascript/jQuery to transform a block of HTML markup into the slider, all customization is done trough the use of HTML5 data attributes and CSS. The markup is valid for HTML5 rules and the content will be fully indexed by search engines. The general structure of markup follows this schema:

 

01 A surrounding DIV is the basic holder for your Slider:

Put it in the HTML:

<div id="services-example-1" class="theme1">...</div>

Important is to give the class theme1 here. You could use our example theme1 as boilerplate/base for your own style adaptions. Just copy&paste the theme1 parts of the settings.css, modify them and change the class in the DIV. So you could easily switch forth and back.

02 An unordered list inside holds the different slides as list elements:

The next chapter describes the inner of this list elements.

It is divided into two parts: A normal/preview view and the detailed view which opens when the "readmore" button is been clicked.

You can basically put in the HTML of your choice in both containers. Remember to use the preview/normal stuff directly in the <li>, the detailed view elements belong into the DIV with the class page-more.

The more advanced users will have no problems in customizing the slider up to every detail and create their own sliders with using the CSS files.

The CSS responsible for the layout of your slider is the settings.css.
